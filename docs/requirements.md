# 项目需求文档

## 项目目标

搭建对话文本标注平台，对用户消息与 AI 助手回复的质量进行人工标注。

## 标注维度

### 1. 有用性判断（必选）

| 标签 | 含义 |
|------|------|
| 有用 | 助手回复正确、相关、有帮助 |
| 无用 | 助手回复存在明显问题 |

### 2. 错误类型（可选，标注"无用"时填写）

| 类型 | 说明 |
|------|------|
| 事实错误 | 回复包含与事实不符的信息 |
| 逻辑矛盾 | 回复中存在自相矛盾的论述 |
| 不相关 | 回复与用户问题无关，答非所问 |
| 格式错误 | 代码格式、排版问题 |
| 其他 | 其他未列出的错误类型 |

## 数据格式

### 导入格式（JSON）

```json
[
  {
    "id": "conv1",
    "user": "用户消息文本",
    "assistant": "助手回复文本"
  }
]
```

### 导出格式（JSON）

标注完成后导出的数据包含原始字段 + 标注结果（`annotations` 数组），每条标注包含 `usefulness` 和可选的 `error_type` 字段。

## 技术栈

- **标注工具**: LabelStudio (label-studio)
- **运行环境**: Python 3.10 (conda)
- **数据交换**: JSON
